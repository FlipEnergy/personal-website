The Homelab V2.0

I had to turn my k8s cluster into a single node because I needed a separate machine to act as a load balancer otherwise my gateway gets confused. Although each k8s service has its own IP from MetalLB, they share the machine network interface mac address and port-forwarding will sometimes send it down to the wrong IP.


Pi-hole DNS nameserver for ad blocking, using upstream dnscrypt-proxy for DNS over TLS. I set all my machines to use this as their primary DNS nameserver.

Surface 3 running HAProxy to foward traffic from internet to my website and VPN in K8s

My server runs Kubernetes with Metallb to assign Ip addresses to services. It's deployed using Helm3 and runs some dashboards and the following:

OpenVPN for home network access and remote ad blocking running in k8s cluster

Folding@home protein folding for science running in k8s cluster

This website is served out of the k8s cluster and auto scales. This is not an invitation to test it out.
